    • Roy · May 11, 2014

      Not sure what fork or branch you are using. I’ll assume 0.9x. Such a feature was not available in the source when I used Anchor, so I had to add it – among other features. I needed the ability to turn Markdown processing off as I had old imported posts that would not show correctly with the Markdown processor. And I didn’t want to dig into the processor itself. Because there have been changes to the source since I moved away from Anchor, many of my posts/forks/branches don’t line up. Check anchor/functions/articles.php. If you have functions called article_markdown and article_html, you should be able to simplify things. Create a custom field for Posts. In your theme, find the line that says: article_markdown. Change it to an if else statement. It should go a little something like… If the custom field is set to no, use article_html. Otherwise, use article_markdown. Hope this helps.
